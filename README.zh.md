你好！👋 我正在开发 Synnia，这是一个开源工具，旨在帮助你在 AI 时代管理创意资产的混乱局面。



## 什么是 Synnia？

Synnia 是一个**数字资产一致性引擎**。

在大多数基于节点的工具（如 ComfyUI）中，节点是*功能*（执行此操作 -> 执行彼操作）。
而在 Synnia 中，节点是**资产**（你的图片、文本、角色）。

你可以在无限画布上放置你的资产，组织它们，并使用“代理”（小助手）来扩展你的项目，同时保持所有内容的一致性。

我不是一个全职的专业开发者，而且我经常发现现有的节点工具让人望而生畏。我开始构建 Synnia，是因为我想要一个直观易用的工作空间——一个学习曲线平缓、邀请你来“玩耍”而不是强制你“学习”的工具。它专为独立开发者、作家和世界构建者设计，帮助他们保持创意宇宙的连贯性。

## 当前状态（目前已完成的工作）

我们刚刚完成了一次大规模的重构（架构 V4）！以下是目前已实现的功能：

-   **以资产为中心的架构**：将“数据”（资产）与“视图”（节点）分离。你的数据独立于画布存在。
-   **图引擎 V4**：一个重写的核心，能够比以前更流畅地处理复杂的交互、拖拽和连接。
-   **智能布局（机架模式）**：灵感来源于音乐制作软件。你可以将节点堆叠到“机架”中，它们会自动调整大小并为你处理布局。不再需要手动对齐方框！
-   **动态配方**：我们有一个 `RecipeNode`，可以根据 JSON Schema 动态生成输入表单。
-   **Tauri + React**：它在你的机器上本地运行。快速且私密。

## 路线图（待办事项清单）

还有很多工作要做。以下是我接下来要进行的工作：

### 1. 更多“乐高积木”（节点）
-   **Markdown 编辑器**：在画布内实现更好的写作和文档管理。
-   **JSON 编辑器**：用于微调原始数据。
-   **调试节点**：用于检查数据流的实际情况。

### 2. 标准厨房（基本配方）
-   目前，你必须从头开始构建所有东西。我希望添加一个“配方”标准库，如 HTTP POST 请求、文本处理工具等，以便你可以立即开始工作。

### 3. 元配方（“应用商店”时刻）
-   **配方组合**：允许你将一个配方 + 另一个配方 + 一些资产组合成一个*新配方*。
-   **分享**：理想情况下，你应该能够将你的工作流导出为单个文件并与他人分享。

### 4. 升级大脑（存储）
-   目前，所有内容都以 JSON 文件的形式保存。它虽然能工作，但不够健壮。
-   **数据库迁移**：迁移到本地数据库（SQLite？）以实现更可靠的数据处理。
-   **虚拟文件系统**：一个用于管理文件、自动生成缩略图和处理大型资产而不会出现延迟的完善系统。
