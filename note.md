  具体交互方案：

  方案：节点的 "右键菜单" + "快捷动作栏"

  当用户在一个 "图片节点" 上点击右键（或悬浮时出现的工具栏）：

   * 一级菜单：
       * 👁️ 预览 / 打开
       * ✨ 呼叫 Agent... (进入对话模式)
       * 🛠️ 工具箱 (Tools)  --> (这里安置基础功能)
       * 🗑️ 删除 / 归档
   * 二级菜单 (工具箱展开)：
       * ✂️ 切图 (Grid Split) -> 点击后直接生成子节点
       * 🧩 抠图 (Remove BG) -> 点击后生成一个去背的新节点
       * 🆙 超分 (Upscale) -> 点击后生成一个高清新节点
       * ☁️ 上传 (Upload) -> 点击后节点状态变为"已上传"，显示链接

  这样的好处：
   1. 高效：这种确定性操作，两步点击就完成了，不需要打字对话。
   2. 清晰：用户知道哪里找灵感（Agent），哪里找工具（右键）。
   3. 一致性：无论是 Agent 生成的，还是工具生成的，结果都会变成一个新的节点，并自动连线。
       * 比如：原图节点 --(Action: RemoveBG)--> 透明底图节点。
       * 这个 Action 同样被记录在连线上，依然享受 "上游变动 -> 下游过期" 的一致性逻辑。




PM 补充建议：自由发挥的空间

  我们需要给用户提供最直接的入口，让他们能像在白板上贴便利贴一样，轻松创建内容。

   1. 拖拽上传 (Drag & Drop):
       * 交互：直接把电脑里的图片、TXT 文件拖进 Canvas。
       * 结果：自动生成对应的 Image Node 或 Text Node。
       * 后台：Synnia 将文件复制/移动到项目文件夹的 assets/ 目录，并在 DB 注册。

   2. 双击创建 (Double-click to Create):
       * 交互：在 Canvas 空白处双击。
       * 结果：弹出一个空的 "便利贴"（通用输入框）。
       * 内容识别：
           * 用户输入一段话 -> 变成 Text Node。
           * 用户粘贴一个 URL -> 变成 Link Node 或自动抓取内容的 Image Node。
       * 手动编辑：用户可以随时双击任何文本节点进行修改（手写 Prompt、修改人设）。

   3. 手动连线 (Manual Linking):
       * 场景：用户拖进来一张自己画的线稿，又拖进来一段自己写的人设。
       * 操作：用户按住线稿节点的把手，拉一根线到人设节点。
       * 语义：建立手动关联（"这张图是参考这段话画的"）。虽然系统无法自动重算，但建立了一致性追踪的路径。
